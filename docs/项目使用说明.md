---

## 1. 项目简介

社区健康档案管理系统是一个基于前后端分离思路实现的课程设计项目。

系统整体结构如下：

- **前端**：HTML + CSS + 原生 JavaScript
- **后端**：C++17 + cpp-httplib + nlohmann/json
- **数据库**：MySQL 8.0 / MariaDB

虽然前后端逻辑上是分离的，但当前项目的实际运行方式是：

- 后端服务不仅提供 API
- 后端还负责托管前端静态页面

所以你不需要额外再起一个前端开发服务器。

只要后端服务启动成功，浏览器直接访问：

```text
http://127.0.0.1:8080
```

即可进入系统登录页。

```text

frontend/  前端页面、脚本、样式
```

你最需要关注的文件有：

## 4. 运行前需要准备什么

无论你是在 Windows 本地还是 WSL 中运行，至少都需要准备以下内容：

### 4.1 必备软件

1. **MySQL 8.0** 或兼容的 MariaDB
2. **CMake**
3. **Git**
4. **支持 C++17 的编译器**
5. **MySQL / MariaDB 客户端开发库**

### 4.2 为什么需要 Git

因为后端构建脚本会通过 CMake 的 `FetchContent` 自动拉取：

- `cpp-httplib`
- `nlohmann/json`

如果你的环境里没有 Git，CMake 可能会在下载依赖时失败。

## 5. 数据库初始化

这一步是所有运行方式都必须做的。

### 5.1 创建数据库

先在 MySQL 中创建数据库：

```sql
CREATE DATABASE IF NOT EXISTS community_health DEFAULT CHARACTER SET utf8mb4;
```

### 5.2 执行 SQL 脚本

建议按下面顺序执行：

1. `sql/init.sql`
2. `sql/views.sql`
3. `sql/procedures.sql`
4. `sql/triggers.sql`
5. `sql/permissions.sql`
6. `sql/seed.sql`（可选，推荐）

如果你使用 MySQL 命令行，可以在项目根目录下执行：

```bash
mysql -u root -p community_health < sql/init.sql
mysql -u root -p community_health < sql/views.sql
mysql -u root -p community_health < sql/procedures.sql
mysql -u root -p community_health < sql/triggers.sql
mysql -u root -p community_health < sql/permissions.sql
mysql -u root -p community_health < sql/seed.sql
```

如果你使用的是图形化工具，例如：

- Navicat
- DBeaver
- MySQL Workbench
- DataGrip

也可以手动打开这些 SQL 文件并依次执行。

## 8. 运行方式一：Windows 本地运行（不使用 WSL）

如果你当前不在 WSL 环境里，而是在普通 Windows 里开发或运行，可以按本节操作。

### 8.1 推荐方案

Windows 下推荐优先使用：

- **MSYS2 + MinGW-w64 + CMake**

原因是当前项目的 CMake 配置更偏向 GCC / Clang 风格，和 MinGW 的兼容性通常更好。

### 8.2 推荐安装内容

建议安装：

- Git
- CMake
- MSYS2
- UCRT64 或 MinGW64 工具链
- Ninja（推荐）
- MariaDB Connector/C 或 MySQL Connector/C

### 8.3 编译前说明

后端依赖 MySQL / MariaDB 的客户端开发库。

如果 CMake 自动识别不到库文件，就要手动指定：

- `MYSQLCLIENT_INCLUDE_DIR`
- `MYSQLCLIENT_LIBRARY`

### 8.4 配置与编译

在项目根目录执行：

```bash
cmake -S backend -B backend/build -G Ninja
cmake --build backend/build -j 4
```

如果自动识别 MySQL 开发库失败，可以改用：

```bash
cmake -S backend -B backend/build -G Ninja \
  -DMYSQLCLIENT_INCLUDE_DIR="你的 include 路径" \
  -DMYSQLCLIENT_LIBRARY="你的库文件路径"
cmake --build backend/build -j 4
```

编译成功后，通常会生成：

```text
backend/build/community_health_server.exe
```

### 8.5 在 PowerShell 中启动

进入项目根目录，执行：

```powershell
$env:DB_HOST = "127.0.0.1"
$env:DB_PORT = "3306"
$env:DB_USER = "ch_admin"
$env:DB_PASS = "Admin@2026!"
$env:DB_NAME = "community_health"
.\backend\build\community_health_server.exe
```

### 8.6 在 cmd 中启动

进入项目根目录，执行：

```cmd
set DB_HOST=127.0.0.1 && set DB_PORT=3306 && set DB_USER=ch_admin && set DB_PASS=Admin@2026! && set DB_NAME=community_health && .\backend\build\community_health_server.exe
```

### 8.7 启动成功的输出示例

```text
[DB] 连接成功: community_health@127.0.0.1:3306
================================================
 社区健康档案管理系统 — 后端服务器
 监听地址: http://0.0.0.0:8080
 API Base: http://localhost:8080/api/v1
================================================
```

看到类似输出后，就可以在浏览器打开：

```text
http://127.0.0.1:8080
```

## 9. 运行方式二：WSL 中运行

如果你已经安装了 WSL，并希望在 Linux 环境中编译和运行项目，可以按本节操作。

### 9.1 适用场景

适合以下情况：

- 你已经装好了 Ubuntu / Debian 等 WSL 发行版
- 你希望用 Linux 工具链编译 C++
- 你更熟悉 Linux 命令行

### 9.2 WSL 中推荐安装的依赖

以 Ubuntu / Debian 为例，可安装：

```bash
sudo apt update
sudo apt install -y build-essential cmake git pkg-config default-libmysqlclient-dev mysql-client
```

如果你使用的是 MariaDB 客户端开发库，也可以装：

```bash
sudo apt install -y libmariadb-dev
```

### 9.3 进入项目目录

假设你的项目在 Windows 的 `E:` 盘：

```bash
cd /mnt/e/CppTrainingProject/DatabaseCourseDesign
```

如果你项目路径不同，请按实际路径修改。

### 9.4 推荐方式：使用 [`Makefile`](Makefile) 与 [`start_linux.sh`](start_linux.sh)

现在项目根目录已经提供：

- [`Makefile`](Makefile)：统一管理编译、启动、停止、状态、日志、健康检查
- [`start_linux.sh`](start_linux.sh)：对 `make` 做了一层更易用的封装

**推荐优先使用这两个入口，而不是手写很长的启动命令。**

### 9.5 先给脚本执行权限

在项目根目录执行：

```bash
chmod +x ./start_linux.sh
```

### 9.6 一键前台启动（推荐演示时使用）

前台启动会占用当前终端，按 `Ctrl + C` 可停止：

```bash
./start_linux.sh run
```

这条命令内部会自动调用：

- `make run`
- 自动配置 CMake
- 自动编译后端
- 自动清理 8080 端口旧进程
- 自动带上数据库环境变量启动服务

### 9.7 一键后台启动（推荐日常开发使用）

如果你希望服务在后台运行：

```bash
./start_linux.sh start
```

启动成功后即可访问：

```text
http://127.0.0.1:8080
```

### 9.8 查看服务状态

```bash
./start_linux.sh status
```

或者直接：

```bash
make status
```

### 9.9 查看健康检查

```bash
./start_linux.sh health
```

或者：

```bash
make health
```

如果服务正常，应返回类似：

```json
{"code":200,"data":{"status":"ok","version":"1.0.0"},"msg":"服务运行正常"}
```

### 9.10 查看后台日志

```bash
./start_linux.sh logs
```

或者：

```bash
make logs
```

后台日志默认保存在：

```text
.run/community_health_server.log
```

### 9.11 停止服务

```bash
./start_linux.sh stop
```

或者：

```bash
make stop
```

### 9.12 重启服务

```bash
./start_linux.sh restart
```

或者：

```bash
make restart
```

### 9.13 如果你仍然想手动编译

也可以继续手动执行：

```bash
cmake -S backend -B backend/build
cmake --build backend/build -j 4
```

编译成功后，通常会生成：

```text
backend/build/community_health_server
```

### 9.14 直接使用 `make`

如果你不想通过脚本，也可以直接在项目根目录执行：

```bash
make run
```

或后台运行：

```bash
make start
```

如果要覆盖数据库参数，可以这样写：

```bash
make start DB_HOST=127.0.0.1 DB_PORT=3306 DB_USER=ch_admin DB_PASS='Admin@2026!' DB_NAME=community_health
```

### 9.15 在 Windows 中通过 WSL 启动

如果你是在 Windows 的 PowerShell 或 `cmd` 中调用 WSL，推荐改为执行脚本：

```cmd
wsl bash -lc "cd /mnt/e/CppTrainingProject/DatabaseCourseDesign && chmod +x ./start_linux.sh && ./start_linux.sh run"
```

如果你希望后台启动：

```cmd
wsl bash -lc "cd /mnt/e/CppTrainingProject/DatabaseCourseDesign && chmod +x ./start_linux.sh && ./start_linux.sh start"
```

相比手写长命令，这种方式更统一，也更适合项目组成员共享使用。

## 10. 如何停止服务

### 10.1 前台运行时停止

如果后端是以前台方式启动的，直接在终端按：

```text
Ctrl + C
```

即可停止。

### 10.2 使用 [`Makefile`](Makefile) 或脚本停止服务

推荐优先执行：

```bash
./start_linux.sh stop
```

或者：

```bash
make stop
```

这会自动：

- 停止 PID 文件记录的后台服务
- 尝试清理 8080 端口占用

### 10.3 WSL 中按端口手动结束进程

如果你需要手动清理 8080 端口，也可以执行：

```bash
fuser -k 8080/tcp
```

然后重新启动服务。

## 12. 常见问题排查

### 12.1 浏览器打不开 `127.0.0.1:8080`

通常是以下原因：

1. 后端没有真正启动成功
2. 8080 端口被占用
3. 启动后立刻退出
4. 数据库连接失败

建议先看终端是否有报错。

---

### 12.2 页面能打开，但登录失败

优先检查：

1. 是否执行过 `sql/init.sql`
2. 默认管理员账号是否存在
3. 环境变量中的数据库连接信息是否正确
4. 当前连接的是不是你预期的那套数据库

默认管理员账号：

- 用户名：`admin`
- 密码：`123456`

---

### 12.3 后端提示数据库连接失败

检查以下项目：

1. MySQL 服务是否启动
2. `DB_HOST` 是否正确
3. `DB_PORT` 是否正确
4. `DB_USER`、`DB_PASS` 是否正确
5. `DB_NAME` 是否正确
6. `sql/permissions.sql` 是否已经执行

有些环境下：

- `127.0.0.1` 可连
- `localhost` 不可连

也有些环境正好相反。

如果遇到问题，可以尝试把：

- `DB_HOST=127.0.0.1`

改成：

- `DB_HOST=localhost`

进行排查。

---

### 12.4 CMake 找不到 MySQL / MariaDB 开发库

这是最常见的问题之一。

解决思路：

1. 确认开发库已安装
2. 重新运行 CMake
3. 手动指定：
   - `MYSQLCLIENT_INCLUDE_DIR`
   - `MYSQLCLIENT_LIBRARY`

示例：

```bash
cmake -S backend -B backend/build -G Ninja \
  -DMYSQLCLIENT_INCLUDE_DIR="你的 include 路径" \
  -DMYSQLCLIENT_LIBRARY="你的库文件路径"
```

---

### 12.5 使用 Visual Studio / MSVC 编译时报参数错误

当前项目构建选项更偏向 GCC / Clang 风格。

如果你直接用 MSVC，可能遇到类似：

- `-Wall` 不识别
- `-Wextra` 不识别

遇到这种问题，建议优先：

1. 使用 **MSYS2 + MinGW**
2. 或者使用 **WSL** 编译

这样更省事。

---

### 12.6 服务启动成功，但页面还是 404

当前项目已经修复为：

- 根路径 `/` 自动跳转登录页
- 后端自动挂载 `frontend/` 静态目录

如果仍然出现 404，优先检查：

1. 你运行的是不是最新编译出的可执行文件
2. 是否重新编译了后端
3. 当前工作目录是否正确
4. 项目中的 `frontend/` 目录是否完整
