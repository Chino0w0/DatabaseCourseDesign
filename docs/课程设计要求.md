# 社区健康档案管理系统 — 课程设计要求与非程序任务

> 本文档记录课程设计的考核要求及程序之外的任务分工（绘图、报告、PPT 等）。

---

## 一、课程设计基本要求

1. 学生自愿分组，3-4 人一组。由组内成员推选小组长并进行分工。
2. 每班相同选题的小组不超过 2 个，配合学委统筹安排。
3. 课程设计时间为 4 周。
4. DBMS 不限（本项目使用 MySQL），开发语言不限（本项目使用 C++ + HTML/JS），系统采用 B/S 模式。
5. 给出数据库设计各阶段的详细设计报告：需求分析、概念结构设计、逻辑结构设计、物理结构设计、软件实现。
6. 数据表设计合理、高效，尽量减少数据冗余，软件界面友好。
7. 提交运行的系统。
8. 写出收获和体会，包括已解决和尚未解决的问题，进一步完善设想与建议。
9. 每个小组进行报告和答辩，讲解设计方案，演示系统运行，汇报分工与合作情况。

---

## 二、课程设计内容（数据库部分）

| 序号 | 内容 | 对应项目产出 |
|------|------|-------------|
| (1) | 系统功能需求分析和数据分析 | 报告第二章 |
| (2) | 数据库需求分析、概念数据库设计，画出局部 & 整体 E-R 图 | 报告第三章 |
| (3) | 逻辑数据库结构设计：实体和联系转化为关系模式，指定主码外码和约束 | 报告第四章 |
| (4) | 数据库定义、数据录入和数据处理 | `sql/init.sql`, `sql/seed.sql` |
| (5) | 用 SQL 实现各关系表，建立索引 | `sql/init.sql` |
| (6) | 设计查询视图 | `sql/views.sql` |
| (7) | 建立存储过程 | `sql/procedures.sql` |
| (8) | 建立触发器，保证数据一致性 | `sql/triggers.sql` |
| (9) | 增删改查操作、创建视图、实验验证 | 报告第八章 + 运行演示 |
| (10) | 建立用户和权限分配，实现数据库安全性 | `sql/permissions.sql` |
| (11) | 根据逻辑/物理设计建立数据库，编制调试应用程序，数据入库并试运行 | 后端 + 前端代码 |
| (12) | 运行数据库应用系统，进行测试 | 测试报告 |

## 四、非程序任务分工

> 以下任务独立于程序开发模块，需单独分配。

### 4.1 绘图任务

| 任务 | 说明 | 建议工具 |
|------|------|---------|
| 数据流图 (DFD) | 系统功能的数据流向 | draw.io / Visio |
| 局部 E-R 图（5 个） | 用户角色、居民社区、居民健康、居民慢性病、随访记录 | draw.io / Mermaid |
| 整体 E-R 图 | 合并局部 E-R 图，消除命名冲突和结构冲突 | draw.io / Visio |
| 系统架构图 | 前后端分离 B/S 架构总览 | draw.io |
| 系统功能模块图 | 按模块划分的功能树 | draw.io |

### 4.2 报告撰写

《课程设计报告》建议大纲：

```
第一章  绪论
  1.1 课题背景
  1.2 系统目标
  1.3 开发环境与技术选型

第二章  系统需求分析
  2.1 系统功能需求分析
  2.2 数据流图 (DFD)
  2.3 数据字典
  2.4 系统功能模块划分

第三章  概念结构设计
  3.1 局部 E-R 图（5 个子图）
  3.2 整体 E-R 图

第四章  逻辑结构设计
  4.1 E-R 图转关系模式
  4.2 关系模式规范化 (3NF 分析)
  4.3 各表主码、外码与约束说明

第五章  物理结构设计
  5.1 存储引擎选择 (InnoDB)
  5.2 索引设计
  5.3 存储结构

第六章  数据库实现
  6.1 建库建表 SQL
  6.2 视图设计与实现
  6.3 存储过程设计与实现
  6.4 触发器设计与实现
  6.5 用户权限管理
  6.6 测试数据导入

第七章  应用系统实现
  7.1 系统总体架构
  7.2 后端实现（C++ 分层架构）
  7.3 前端实现
  7.4 前后端联调

第八章  系统测试
  8.1 测试环境
  8.2 功能测试用例与结果
  8.3 SQL 验证（增删改查 + 视图 + 存储过程 + 触发器）

第九章  收获与体会
  9.1 各成员心得
  9.2 已解决的问题
  9.3 尚未解决的问题
  9.4 进一步完善设想与建议

附录
  附录A  完整 SQL 脚本
  附录B  API 接口文档
  附录C  参考文献
```

### 4.3 答辩 PPT

建议包含以下内容（10 分钟讲解）：

1. 项目概述与分工（1 分钟）
2. 数据库设计：E-R 图 → 关系模式 → 3NF（2 分钟）
3. 数据库实现：视图 / 存储过程 / 触发器 / 权限（2 分钟）
4. 系统架构与关键代码（2 分钟）
5. 系统演示（2 分钟）
6. 总结与收获（1 分钟）

## 五、考核提醒

> **严禁抄袭**：如果发现最后的设计基本相同者（系统需求分析与功能设计、数据库的概念设计、逻辑设计，数据库的实现与运行等内容基本相同），一经验收教师认定其抄袭行为，则小组所有人成绩均为 0 分，且记为作弊。
